## 4. 中间遇到的问题与处理

### 4.1 `uv sync` 下载卡住

- **处理**：项目 `pyproject.toml` 配置清华源 `[[tool.uv.index]]` + `link-mode = "copy"`。

### 4.2 PowerShell 回写破坏 UTF-8

- **现象**：`stream did not contain valid UTF-8`。  
- **处理**：避免用默认编码 `Set-Content` 改 toml；成员包描述改为英文后重写。

### 4.3 Postgres 认证

- 初版默认密码 `postgres` 失败。  
- 用户提供真实密码与库名 `mcp_agent_db` 后写入 `.env`，连通成功。

### 4.4 Memurai 安装（需管理员）

- **目标路径**：`E:\AI\Memurai`  
- **安装包**：Memurai Developer 4.1.2 MSI  
  `https://dist.memurai.com/releases/Memurai-Developer/4.1.2/Memurai-Developer-v4.1.2.msi`
- **非管理员**直接 `msiexec` → 退出码 **1603**（临时目录权限 / 端口检查自定义动作失败）。  
- **处理**：UAC 提权后静默安装：

```powershell
msiexec /i Memurai-Developer-v4.1.2.msi /quiet /norestart `
  INSTALLFOLDER=E:\AI\Memurai `
  ADD_INSTALLFOLDER_TO_PATH=1 INSTALL_SERVICE=1 PORT=6379 ADD_FIREWALL_RULE=1
```

- 安装后服务 `Memurai` 为 Running；`memurai-cli.exe ping` → `PONG`。

### 4.5 PGVector for PostgreSQL 18（需管理员）

- **归档目录**：`E:\AI\PGVector_18`（保留 zip 解压内容便于重装）  
- **二进制来源**（非官方预编译，社区 Windows 包）：  
  [andreiramani/pgvector_pgsql_windows `0.8.3_18.4`](https://github.com/andreiramani/pgvector_pgsql_windows/releases/tag/0.8.3_18.4)  
  文件：`vector.v0.8.3-pg18.zip`
- **步骤**：
  1. 解压到 `E:\AI\PGVector_18`
  2. **停止**服务 `postgresql-x64-18`
  3. 将 `lib\vector.dll`、`share\extension\*`（及可选 `include\...`）拷入 `E:\AI\postgreSQL\install\` 对应目录  
     （无管理员时 Copy 会 Access Denied）
  4. 启动 Postgres
  5. 在 `mcp_agent_db` 执行：`CREATE EXTENSION IF NOT EXISTS vector;`
- **结果**：`vector` 扩展版本 **0.8.3**。

### 4.6 Windows 控制台 GBK

- `check_local_deps` 增加安全打印，避免中文错误信息触发 `UnicodeEncodeError`。
